双模式

product-gtm

产品推广

扫描产品项目文件（README、文档、代码），自动理解产品定位与卖点，生成多角度推广文案，支持小红书与微信公众号双平台输出。

F1 产品理解 → F2 策略 → F3 文案(XHS/MP) → F4 封面 → F5 发布 → F6 追踪
personal-log

个人日志 / 个人 IP

读取 Obsidian 等个人日志，提炼人设特质与内容方向，生成有「真实经历质感」的个人 IP 内容。内置脱敏闸门保护隐私，支持双平台发布。

F1 日志理解 → F2 选题 → F3 文案(XHS/MP) → F4 封面 → F5 脱敏 → F6 发布 → F7 追踪

功能大盘

product-gtm 产品推广

F1
产品理解引擎
读取产品项目文件（README、文档、规则等），输出结构化的产品理解报告：定位、功能、受众、卖点。
Skillproduct-gtm已完成
F2
内容策略规划
基于产品理解，从 6 种角度中规划多篇不同切入点的内容大纲，自动去重已用角度。
Skillproduct-gtm已完成
F3
小红书文案生成
生成适配小红书调性的完整文案：标题(≤20字) + 正文(≤1000字) + 标签(≥5个)，支持逐篇微调。
Skillproduct-gtm已完成
F3-MP
公众号文案生成
生成微信公众号长文格式：标题(≤32字) + 摘要 + 结构化正文（含小标题）+ 标签。独立 Prompt，无 emoji 堆砌。
Skillproduct-gtm已完成

personal-log 个人日志

F1
个人日志库理解
读取 Obsidian 等 Markdown 日志，提炼写作习惯、子场景分布、高频主题、叙事风格，生成人物与叙事档案。
Skillpersonal-log已完成
F2
多场景归类与选题
将日志素材归类为 growth / travel / reflection / life 四种子场景，规划选题大纲并自动避免素材重复。
Skillpersonal-log已完成
F3
真实感文案生成（小红书）
第一人称个人叙事，保留情绪与转折，用模糊化事实代替编造。像朋友分享生活而非广告。
Skillpersonal-log已完成
F3-MP
真实感文案生成（公众号）
个人叙事风格的公众号长文：内省式标题 + 结构化成长叙事正文 + 摘要字段，适合深度读者。
Skillpersonal-log已完成
F5
脱敏与可发布性闸门
逐篇检查隐私风险（真名、公司、地址、财务等 8 项），输出脱敏检查表，用户明确确认后才可发布。
Skillpersonal-log已完成

双模式共享

F4
封面图生成
根据文案角度/子场景生成封面图（3:4 竖版 / 1:1 方形）。product-gtm 有 6 种风格映射，personal-log 有 4 种。
Skill双模式共用已完成
F5/6
多平台发布
三种发布方式：prepare 手动（最安全）、Chrome DevTools MCP 自动化（推荐，操控真实 Chrome）、Playwright legacy。支持小红书 + 公众号双平台。
MCPCLI双模式共用已完成
F6/7
历史内容追踪
product-gtm 追踪角度使用（history.json），personal-log 追踪素材日期与子场景（history-personal.json），自动避免重复。
CLISkill双模式共用已完成

发布方式

三种发布方式按推荐程度排列，各有适用场景。

方式一 · 最安全

prepare + 手动发布

CLI 将内容格式化输出到终端 + 复制到剪贴板，用户在平台 App/网页中手动粘贴发布。零账号风险，适合谨慎场景。

方式二 · 推荐 · 公众号已固化

CDP 自动化（publish -p mp）

以调试模式启动 Chrome（--remote-debugging-port=9222），CLI 通过 Playwright CDP 连接已登录的真实 Chrome，自动填写标题/正文/摘要/封面图并保存草稿。登录状态持久化，后续无需重复登录。

方式三 · legacy

Playwright 自动化（仅 xhs）

独立 Chromium 实例，需单独登录。平台可能检测到自动化操作（navigator.webdriver 等特征）。仍可用，但安全性低于 CDP 方案。

CDP 公众号自动发布工作原理（publish -p mp）

sequenceDiagram
    participant U as 用户
    participant CLI as autosome CLI
    participant PW as Playwright CDP
    participant C as 已登录的 Chrome

    U->>CLI: publish -p mp --post <path>
    CLI->>CLI: check_cdp_available(9222)
    CLI->>PW: connect_over_cdp(localhost:9222)
    PW->>C: 连接已登录的 Chrome 实例
    CLI->>C: 点击「文章」→ 打开编辑器
    CLI->>C: fill 标题
    CLI->>C: keyboard.insert_text(正文)
    CLI->>C: 点击文章设置 → fill 摘要
    CLI->>C: set_input_files(cover.png)
    CLI->>C: 点击「保存为草稿」
    C-->>CLI: 草稿已保存
    CLI->>U: ✅ 草稿已保存，请在手机预览后群发

小红书 vs 公众号发布差异

维度小红书微信公众号
登录方式 账号密码 / 扫码 微信扫码（MCP 方案复用已登录 Chrome）
内容格式 短文（≤1000字）+ 多图 + 标签 长文 + 封面图 + 摘要（digest）
默认发布动作 直接发布 保存草稿（用户手机预览后再群发）
推荐自动化方式 Playwright legacy（publish -p xhs） CDP（publish -p mp） — 代码已固化
Prompt 文件（Agent 对话发布） browser-publish-xhs.md browser-publish-mp.md
跨平台搬运 CLI prepare -p mp --post <xhs目录> 将小红书帖子适配为公众号格式

CLI 操作手册

📦 安装

安装 Python 依赖（Chrome DevTools MCP 由 Cursor 自动加载，无需额外安装）
pip install -r requirements.txt
部署 Cursor Skill（符号链接）
ln -sf "$(pwd)" ~/.cursor/skills-cursor/auto-so-me
[可选] 安装 Playwright（仅 legacy publish 命令需要）
playwright install chromium

📋 内容准备（推荐 · 最安全）

小红书：准备单篇（标题复制到剪贴板 + 终端输出完整内容）
python -m autosome prepare \
  --post ".auto-so-me/campaigns/<batch>/xhs-output/<post-name>"
小红书：准备单篇 + 复制正文（含标签）到剪贴板
python -m autosome prepare \
  --post "<path>" --copy-body
微信公众号：准备单篇（输出标题 + 摘要 + 正文 + 封面图路径）
python -m autosome prepare -p mp \
  --post ".auto-so-me/campaigns/<batch>/mp-output/<post-name>" --copy-body
跨平台搬运：将小红书帖子适配为公众号格式
python -m autosome prepare -p mp \
  --post ".auto-so-me/campaigns/<batch>/xhs-output/<post-name>" --copy-body
批量准备，指定第几篇
python -m autosome prepare \
  --dir "<path>" --index 1

🌐 CDP 自动发布 — 微信公众号（推荐）

💡 Chrome 使用须知
调试 Chrome（--user-data-dir=~/.chrome-debug-profile）是独立的配置，与你日常使用的主 Chrome 完全隔离，主 Chrome 的登录账号、标签页、历史记录不受任何影响。

⚠️ 两者不能同时从 Dock 图标打开：调试 Chrome 运行时，点 Dock 图标会切换到调试窗口而非主 Chrome。
推荐的工作节奏：
  日常工作 → 主 Chrome（Dock 打开）
  需要发布 → Cmd+Q 关闭主 Chrome → 运行下方命令启动调试 Chrome → 发布完毕 → Cmd+Q 关闭调试 Chrome → 从 Dock 重新打开主 Chrome
推荐：添加 shell 别名，方便每次快速启动（加入 ~/.zshrc）
echo 'alias chrome-debug="/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir=$HOME/.chrome-debug-profile"' >> ~/.zshrc && source ~/.zshrc
Step 1：首次启动调试 Chrome（一次性，登录状态持久化）
chrome-debug
# 弹出的 Chrome 中登录 https://mp.weixin.qq.com（仅首次需要）
Step 2：发布单篇公众号帖子（CDP 自动填写 + 保存草稿）
python -m autosome publish -p mp \
  --post "<项目根>/.auto-so-me/campaigns/<batch>/mp-output/<post-name>" \
  --project "<项目根>"
验证 CDP 端口是否就绪
curl http://localhost:9222/json/version
指定自定义 CDP 端口
python -m autosome publish -p mp --post "<path>" --cdp-port 9223

内容角度 / 子场景体系

product-gtm：6 种内容角度

Agent 从 6 种标准角度中为每篇文案选取切入点，自动去重，角度池耗尽后切换受众视角。小红书与公众号均支持同一套角度体系。

角度名称内容结构示例 Hook
pain-point 痛点切入 痛点共鸣 → 解决方案 → 产品介绍 → CTA "写 PRD 写到崩溃的产品经理，看过来"
scenario 场景展示 场景描述 → 使用过程 → 前后对比 → CTA "今天用 AI 帮我做了一整个版本的需求"
comparison 对比测评 传统痛点 → 新方案 → 效率对比 → CTA "手写 PRD vs AI 写 PRD，效率差了 10 倍"
tutorial 教程干货 步骤拆解 → 操作演示 → 要点总结 → CTA "保姆级教程：用 Cursor 搭建 AI 产品经理"
showcase 成果展示 成果展示 → 制作过程 → 产品揭秘 → CTA "这份交互原型 3 分钟就生成了"
story 个人故事 背景 → 遇到问题 → 尝试方案 → 成果 "一个 PM 如何用 AI 把效率翻了 3 倍"

personal-log：4 种子场景

日志素材按子场景归类，同一批次尽量不重复，新批次自动避免短期内消费同一素材。

子场景名称内容倾向封面风格
growth 成长 学习、习惯、职业思考、复盘 文字卡片式，蓝绿渐变
travel 游玩出行 行程体验、城市印象、轻量化攻略 风景意境式，暖色/自然色
reflection 日常思考 观点、情绪、价值观侧写 极简文字式，莫兰迪色
life 其它生活 未归入上类的可分享日常 生活感插画式，奶油色/暖黄
