# 浏览器自动化发布 — 微信公众号

通过 Chrome DevTools MCP 工具在用户的真实 Chrome 浏览器中完成微信公众号文章发布。

## 前置条件

- Chrome DevTools MCP 已配置并可用
- 用户的 Chrome 浏览器中已登录微信公众号后台（https://mp.weixin.qq.com）
- 帖子的 `content.md` 已生成且经用户确认

## 发布流程

### Step 1: 读取帖子数据

从 `content.md` 中提取：
- `title`：标题
- `digest`：摘要（如无则取正文前 120 字）
- `body`：正文（Markdown 格式）
- `author`：作者（如有）
- 同目录下的图片文件（cover.* 作为封面图）

### Step 2: 打开图文编辑器

```
使用 navigate_page 打开 https://mp.weixin.qq.com
```

等待页面加载完成。如果需要登录，截图通知用户。

登录后，导航到「内容与互动」→「图文消息」→「写新图文」，或直接找到「+ 创作」按钮。

使用 `take_screenshot` 确认当前页面状态，动态判断导航路径。

### Step 3: 填写标题

找到标题输入框（通常是页面顶部的大输入框，placeholder 可能包含"请在这里输入标题"）。

使用 `click` 点击后 `type_text` 输入标题。

### Step 4: 填写正文

公众号编辑器是富文本编辑器。找到正文编辑区域（contenteditable 的 div）。

使用 `click` 点击编辑区，然后 `type_text` 输入正文内容。

由于公众号正文支持富文本，如果 Markdown 格式的正文中包含小标题（`##`），可以尝试：
- 先输入全部文本
- 或分段输入，每段之间适当停顿

### Step 5: 设置封面图

找到封面图上传区域，使用 `upload_file` 上传 cover 图片。

公众号封面图通常在编辑器右侧或底部的设置面板中。使用 `take_screenshot` 定位具体位置。

### Step 6: 填写摘要

找到摘要/描述输入区域（可能标注为"摘要"或在封面图设置附近），输入 digest 内容。

### Step 7: 截图确认

使用 `take_screenshot` 截取当前页面，展示给用户确认：
- 标题是否正确
- 正文是否完整
- 封面图是否设置
- 摘要是否填写

**必须等待用户明确确认「可以发布」后，才能执行下一步。**

### Step 8: 发布或保存草稿

用户确认后，根据用户指示：
- **群发**：点击「群发」按钮（这会推送给所有关注者，需慎重）
- **保存草稿**：点击「保存为草稿」（推荐，用户可在手机微信中预览后再群发）
- **预览**：如有预览功能，先发送预览到用户微信

发布/保存后使用 `take_screenshot` 确认操作结果。

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 未登录 | 截图告知用户需要扫码登录，等待用户完成登录后重试 |
| 编辑器布局不确定 | 先用 `take_screenshot` 截图分析，再决定操作步骤 |
| 富文本格式丢失 | 告知用户可在公众号后台手动调整格式 |
| 封面图上传失败 | 截图告知用户，请手动上传 |
| 找到多个编辑区域 | 截图确认并询问用户 |

## 安全原则

- **必须**在发布/群发前截图让用户确认
- **推荐**默认保存为草稿而非直接群发
- **禁止**在用户未确认的情况下自动点击群发
- personal-log 模式必须先通过脱敏闸门（`desensitization_confirmed: true`）
- 公众号群发不可撤回，Agent 应额外提醒用户确认

## 与小红书发布的区别

| 维度 | 小红书 | 微信公众号 |
|------|--------|-----------|
| 登录方式 | 账号密码/扫码 | 微信扫码 |
| 内容格式 | 短文 + 多图 + 标签 | 长文 + 封面图 + 摘要 |
| 发布方式 | 直接发布 | 群发（不可撤回）或保存草稿 |
| 编辑器 | 简单输入框 | 富文本编辑器 |
| 推荐默认操作 | 发布 | 保存草稿（用户手机预览后再群发） |
